# 可行性计划：区块链与 AI 在 SkyTrust 的落地实现

## TL;DR
- 目标是在保护隐私与合规的前提下，落地区块链证据链（身份、飞行日志、设备护照、押金与理赔）以及 AI 的安全性、调度与智能决策能力，形成一个可落地的 MVP（最小可行产品）。优先考虑私有/联盟链场景，必要时保留公有链方案的可选性。
- 本计划提供两条主线：底层网络选型与 MVP 边界，并给出阶段性里程碑、风险及验证方法。

## 背景与目标
- SkyTrust 的区块链应用要解决无人机身份、飞行日志、设备全生命周期、押金/保险、调度及风控等关键场景的证据可信性与可追溯性，同时通过 AI 提升安全性与调度效率。
- 目标：在最小可控成本下，以 MVP 覆盖核心信任与安全能力，并为后续扩展提供清晰的技术路线与风险控制点。

## 可选方案（高层对比）
- 方案 A：私有/联盟链 + 设备 NFT（护照）+ 事件哈希锚定
  优点：低延迟、可控治理、隐私保护更好
  风险：搭建成本、联盟治理成本、合规挑战
  MVP 要点：身份哈希、飞行日志哈希、设备 NFT 基础元数据、押金合约框架

- 方案 B：公有链 + 私有侧链/二层（L2）
  优点：公链安全性与透明度，扩展性通过 L2 实现
  风险：隐私、成本波动、跨链治理
  MVP 要点：对敏感数据使用哈希锚定，核心证据落在侧链/锚定在公链

- 方案 C：混合方案（底层私有/联盟链、对外公链锚定）
  优点：平衡隐私与透明度
  风险：架构复杂度、跨链一致性
 MVP 要点：分层数据流、核心证据在私网，关键指纹锚定在公链

> 你可以选择一个偏好方向，我再据此细化 MVP 边界与实施路径。

## MVP（落地边界）
- 区块链 MVP：
  - 身份哈希与资质证据锚定（不可公开隐私信息，仅哈希/授权的可验证性）
  - 飞行日志哈希锚定（核心事件哈希）
  - 设备数字护照（基础 NFT，包含维修/固件等元数据）
  - 押金与保险的初步智能合约框架（核心流程）
- AI MVP：
  - 起飞前 AI 预检与授权（简单的视觉/传感数据检查）
  - 实时地理围栏的简化实现（本地化或边缘计算）
  - 动态定价的初版模型（基于历史数据的简单预测）
  - 生成式客服/教学助手的初步接入（模板驱动，低成本实现）

## 架构要点（要点化）
- 数据流：前端/后端 API -> 区块链合约/哈希锚定 -> 链上证据 -> off-chain 存储/对照表
- 数据分层：敏感数据走 off-chain，哈希锚定用于不可抵赖性
- 安全与合规：私钥管理、审计、合约审计、密钥轮换机制
- 监控与可观测性：证据可查询、AI 调用可追溯、异常检测日志

## 风险与缓解
- 隐私保护不足：采用哈希锚定 + 最小上链数据策略
- 性能/成本：私有/联盟链 MVP 降低延迟，公有链采用 L2 方案
- 安全：合约审计、密钥管理、灾备设计
- 法规合规：隐私、数据保留期、数据主权等合规点

## 时间线与里程碑（初步）
- 第 0-2 周：收集需求、对比方案、评估风险
- 第 3-6 周：搭建 MVP 架构原型（私有/联盟链基础、NFT Passport 草案、哈希锚定初步实现）
- 第 7-12 周：AI MVP 原型与区块链 MVP 集成
- 第 13-16 周：集成测试、仿真、灰度发布、监控与安全评估

## 验收标准（核心/可度量）
- 区块链 MVP 验证点：身份哈希、飞行日志哈希、设备 NFT 基础元数据可写入/查询、押金合约基本流程
- AI MVP 验证点：核心能力可用、重要场景可执行、基本安全告警可用
- 性能/成本基线：锚定成本、查询延迟、写入时延符合目标
- 安全与合规基线：完成初步安全审计与隐私评估
